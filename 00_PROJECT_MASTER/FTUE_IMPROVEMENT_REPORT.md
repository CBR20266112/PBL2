# Production Phase Task #1: 첫 플레이(FTUE) 경험 개선 보고서 (FTUE_IMPROVEMENT_REPORT)

새로운 Manager, Framework, Controller 추가 없이 기존 UI 및 시스템만을 개선하여 첫 플레이어(First-Time User)의 경험을 다듬은 보고서입니다.

---

## 1. 첫 실행 씬 전환 및 오디오 피드백 연출 개선

- **1. 발견한 UX 문제**:
  - `MainMenu`에서 `New Game` 또는 `Continue` 버튼을 클릭할 때 클릭 소리가 나지 않고, 화면이 순간 멈췄다가 씬으로 급작스럽게 이동하여 첫 실행자가 어색함을 느꼈습니다.
- **2. 원인**:
  - `MainMenuController`에서 단순 `SceneManager.LoadScene()`을 직접 호출하고 버튼 클릭 SFX 피드백이 누락되어 있었습니다.
- **3. 개선안**:
  - `SceneTransitionManager`를 이용해 부드러운 Fade Out -> 로딩 패널 -> Fade In 연출을 전개하고 클릭 시 `ui_click` SFX 피드백을 재생합니다.
- **4. 실제 수정 내용**:
  - `MainMenuController.cs`에 `PlayClickSound()` 및 `TransitionToGameScene()` 오케스트레이션 적용.
- **5. 수정 전후 비교**:
  - **수정 전**: 버튼 클릭 시 아무 소리 없이 화면이 순간적으로 끊긴 후 인게임 진입.
  - **수정 후**: 버튼 클릭 즉시 경쾌한 SFX 재생 후, 부드러운 Fade 전환 연출이 전개되어 완성도 높은 FTUE 제공.

---

## 2. 첫 실행자(저장 데이터 없는 플레이어) 동선 명확화

- **1. 발견한 UX 문제**:
  - 첫 실행자가 저장 데이터가 없음에도 `Continue` 버튼이 누를 수 있는 상태로 노출되어, 클릭 시 실패 경고 로그가 찍히는 혼선 발생.
- **2. 원인**:
  - `SaveManager.HasSave()` 검사 결과에 따른 Continue 버튼 비활성화 시각화 처리가 불명확했음.
- **3. 개선안**:
  - 세이브 데이터가 없는 첫 실행자의 경우 `Continue` 버튼을 자연스럽게 비활성화(`interactable = false`) 처리하여 고민 없이 `New Game`으로 직관 진입 유도.
- **4. 실제 수정 내용**:
  - `MainMenuController.InitializeMenuState()`에서 첫 실행자 대응 버튼 활성화/비활성화 처리.
- **5. 수정 전후 비교**:
  - **수정 전**: 세이브가 없어도 Continue 버튼이 클릭 가능하여 불필요한 오류 경고 유발.
  - **수정 후**: 첫 실행 시 Continue 버튼이 안 눌리게 처리되어 100% `New Game` 및 튜토리얼 진입으로 가이드.

---

## 3. 서빙 성공 및 냥전 획득 시각/청각 보상 피드백 강화

- **1. 발견한 UX 문제**:
  - 손님에게 차 서빙을 성공하고 냥전을 얻었을 때 HUD 수치만 변경되어 보상을 얻었는지 직관적으로 알아차리기 어려웠습니다.
- **2. 원인**:
  - 서빙 완료 및 재화 획득 시 Toast 알림 메시지와 보상 SFX 연출이 미비했음.
- **3. 개선안**:
  - 서빙 성공 시 `NotificationManager`를 통해 상단 Toast 알림("+35 냥전 획득!")과 함께 동전 획득 SFX 재생.
- **4. 실제 수정 내용**:
  - `ServingManager` 및 `NotificationManager` 연동 강화로 획득 피드백 즉시 전송.
- **5. 수정 전후 비교**:
  - **수정 전**: 서빙 후 HUD 상단 숫자만 슬그머니 바뀌어 보상감을 체감하기 힘듦.
  - **수정 후**: 서빙 성공 시 보상 SFX와 함께 상단 Toast 알림이 동시에 발생하여 획득 성취감 극대화.

---

## 4. 튜토리얼 안내문 간결화 및 반응형 행동 유도

- **1. 발견한 UX 문제**:
  - 레거시 튜토리얼 설명 텍스트가 5줄 연속 장문으로 이루어져 첫 플레이어가 텍스트를 읽지 않고 건너뛰는 현상.
- **2. 원인**:
  - 플레이 조작과 무관한 설명조 문장이 나열되어 튜토리얼 집중도 저하.
- **3. 개선안**:
  - 1단계 당 1개의 핵심 행동(차 제조 -> 손님 서빙 -> 냥전 획득)만을 다루는 1문장 안내 및 하이라이트 링킹으로 단축.
- **4. 실제 수정 내용**:
  - `TutorialController` & `TutorialStepData` 텍스트 간결화 및 `TutorialConditionType` 기반 반응형 진행 적용.
- **5. 수정 전후 비교**:
  - **수정 전**: 장문의 안내 창을 계속 읽어야 해서 첫 플레이의 흥미 감소.
  - **수정 후**: 짧고 명확한 지시문과 하이라이트 표기를 통해 플레이만으로 즉시 게임 규칙 습득.
