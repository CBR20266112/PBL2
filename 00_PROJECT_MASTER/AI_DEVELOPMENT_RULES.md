## 6. Editor Setup Tool 실행 정책

모든 Tools > Setup 계열 에디터 툴은 **사용자가 명시적으로 실행**하는 것을 기본 정책으로 합니다.

- InitializeOnLoad 또는 EditorApplication.delayCall을 통한 자동 실행을 사용하지 않습니다.
- Setup Tool은 **Idempotent** 하게 동작합니다. 여러 번 실행해도 기존 에셋을 덮어쓰지 않습니다.
- 에셋이 이미 존재하면 스킵하고, 없는 것만 신규 생성합니다.
- **기존 에셋을 수정하거나 덮어쓰지 않습니다.** 기존 에셋을 변경해야 하는 경우에는 별도의 `Tools > Migration` 또는 `Tools > Upgrade` 기능으로 제공합니다.
- 실행 결과는 Console 로그와 Dialog로 명확히 보고합니다.

## 7. 프로젝트 공통 비즈니스 규칙

앞으로의 모든 시스템 구현 시 다음 규칙을 기본 기준으로 준수하여 개발합니다.

1. **기본 재화 규격 ("냥전")**:
   - 모든 인게임 가격은 '냥전' (int) 단위를 사용합니다. 소수점 단위는 사용하지 않습니다.
   - 현실 화폐 단위(원, 엔, 달러 등)는 코드나 에셋, 로그 메시지 내에서 직접 사용하지 않습니다.
2. **유료 재화 규격 ("보석(Gem)")**:
   - 향후 확장을 고려하되, 이번 단계에서는 실제 구현을 진행하지 않고 데이터 구조적으로 확장 가능한 구조만 염두에 둡니다.
3. **다국어(Localization) 설계**:
   - 한국어와 영어를 기본 지원 대상으로 하며, 코드 내부에 유저에게 보여지는 UI 문자열을 직접 하드코딩하지 않습니다.
   - 재화 이름 등의 표현도 Localization Key를 사용할 수 있는 구조로 구성합니다.
4. **가격 밸런스 스케일 기준**:
   - 일반 음료: 약 20~40 냥전
   - 현대 음료: 약 30~60 냥전
   - 재료 가격 스케일 역시 위 범위와 조화를 이루도록 기준점을 유지합니다.
   - 밸런스 튜닝이 용이하도록 하드코딩을 지양하고 Inspector 또는 Database 기반으로 가격 데이터를 관리합니다.
5. **손님 보상 구조**:
   - 판매 성공 시 '냥전' 획득 처리합니다.
   - 만족도에 따라 경험치(XP) 및 친밀도(Affinity/Friendship) 소량을 획득할 수 있도록 하되, 이번 단계에서는 실제 연동하지 않고 설계 구조상 훅(Hook)이나 가용 가능한 인터페이스의 형태만 열어둡니다.

## 8. 가구(Furniture) 배치 시스템 설계 지침

가구 배치 시스템의 장기 아키텍처 기준입니다. Furniture 관련 모든 시스템은 이 지침을 따릅니다.

### 8.1 슬롯(Slot) 기반 배치 원칙
- **자유 배치 금지**: 유저가 화면 아무 곳에나 드래그하여 배치하는 자유 배치(Free Placement)는 구현하지 않습니다.
- **슬롯 기반 배치만 허용**: 씬(Scene)마다 미리 정의된 고정 슬롯(Slot) 위치에만 가구를 배치할 수 있습니다.
- **좌표 미사용**: 가구의 좌표(Vector3)는 저장하지 않습니다. 저장해야 하는 정보는 "어느 슬롯에 어떤 가구가 배치되었는가" (`slotId → furnitureId`) 관계입니다.

### 8.2 FurnitureType 및 SlotType 매칭
- `FurnitureData`에는 `FurnitureType` (예: `Chair`, `WallDecoration`, `Table`, `Plant` 등) 필드를 포함합니다.
- `FurnitureSlot`에는 `SlotType` 필드를 포함합니다.
- 배치 가능 여부 판정: `SlotType == FurnitureType` 일치 시에만 해당 슬롯에 가구를 배치할 수 있습니다.
- 동일 타입이 아닌 가구는 슬롯에 배치할 수 없습니다.

### 8.3 슬롯 구조 예시
```
WallDecoration_A, WallDecoration_B, WallDecoration_C
FloorDecoration_A, FloorDecoration_B
Chair_A, Chair_B, Chair_C, Chair_D
Table_A, Table_B, Table_C
Shelf_A, Shelf_B
Plant_A, Plant_B
Window_A, Window_B
Counter_A (고정), TipBox_A (고정), MenuBoard_A (고정), SignBoard_A (고정), Door_A (고정)
```

### 8.4 SaveData 구조
- `slotId → furnitureId` 관계를 `List` 또는 `Dictionary` 형태로 직렬화하여 저장합니다.
- 좌표(Vector3), 회전(Quaternion) 정보는 저장하지 않습니다.

### 8.5 FurnitureManager의 책임 범위 (제한 사항)
- **FurnitureManager는** 좌표 계산, 드래그 처리, 충돌 검사, 회전 기능을 수행하지 않습니다.
- 시각적 표현(씬 내 스프라이트/프리팹 표시)은 `FurniturePlacement` (또는 Presenter/View) 계층이 슬롯 위치에 맞추어 Sprite 또는 Prefab을 렌더링합니다.
