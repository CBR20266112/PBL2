# MASTER CROP PLAN

> **작성 일자**: 2026-07-12
> **문서 목적**: 분석된 10종의 에셋 시트를 Unity Sprite로 가장 효율적이고 오류 없이 추출하기 위한 마스터 실행 계획
> **도구 스택**: Python (Pillow, OpenCV, SciPy), Photoshop/GIMP(수동)

## 1. 크롭 전략 (자동 vs 수동)

총 10개의 PNG 파일을 자동화 가능성에 따라 3가지 전략 그룹으로 분류합니다.

### 🟢 그룹 A: 완전 자동 크롭 (Alpha Bounding Box)
가장 우선적으로 스크립트를 통해 일괄 처리할 대상입니다. 배경이 완전 투명하고 간격이 일정합니다.
- **대상 파일**:
  - `7273a87f` (가구/소품 카탈로그)
  - `7fc5059d` (가구/인테리어 시트)
  - `adf7adab` (Tea & UI 종합 아이콘 시트)
  - `b61d0da6` (건축 구조 요소 시트)
  - `f72e7bd5` (UI 패널 & 시스템 아이콘 시트)
  - `fed646b9` (건축 구조/인테리어 부품 시트)
- **추출 방식**: 
  - `OpenCV findContours` 또는 `Pillow getbbox`로 투명 배경(Alpha) 기준 바운딩 박스 자동 감지
  - 노이즈 필터링 (최소 크기 20x20 픽셀 적용)
  - 자동 감지 후 10px 여백(Padding)을 더해 개별 파일 저장

### 🟡 그룹 B: 반자동 크롭 (수동 ROI + 자동 추출)
배경이 불투명하거나 간격이 일정하지 않아, 큰 구역을 사람이 지정해주고 내부는 코드가 처리합니다.
- **대상 파일**:
  - `c06ec44a` (게임 종합 레퍼런스 - 영문)
  - `f6ceccc7` (게임 종합 레퍼런스 - 한글)
  - `f837fa6a` (브루잉 애니메이션 시퀀스)
- **추출 방식**:
  - 스크립트에 섹션 경계 좌표(ROI) 또는 행(Row)별 Y좌표 범위를 입력
  - 각 구역 내부에서 흰 배경(Color Threshold) 또는 투명도 기준으로 자동 추출
  - 애니메이션 시퀀스는 프레임간의 간격을 고정된 폭(Width)으로 그리드 분할할 수도 있음

### 🔴 그룹 C: 완전 수동 마스킹 및 분리
코드 자동화 시 오류율이 90% 이상인 복잡한 합성 씬과 이펙트들입니다. 그래픽 툴을 이용해 사람이 직접 분리해야 합니다.
- **대상 파일 및 오브젝트**:
  - `25459862` (풀스크린 UI 목업 씬 전체)
  - 브루잉 시트(`f837fa6a`)의 수증기(Steam) 이펙트 등 반투명 경계가 모호한 픽셀
  - 다른 오브젝트에 걸쳐 있는 차판 이펙트나 겹친 구조물(기둥과 보)
- **추출 방식**: Photoshop이나 GIMP를 사용해 레이어 마스킹 작업 후 수동 슬라이싱.

## 2. 작업 실행 우선순위 (우선순위 로드맵)

Unity 에셋 제작의 효율을 위해 아래 단계별로 진행합니다.

### Phase 1: 자동화 기반 대량 추출 (전체 에셋의 70% 확보)
- **실행 대상**: 그룹 A (투명 배경 6종 시트)
- **우선 항목**: 바닥재, 벽, 창문, 다구, 기본 가구, UI 아이콘
- **작업**: Python 자동 크롭 스크립트를 작성하여 일괄 실행 후 검수. 1시간 내 200여 개 에셋 확보 가능.

### Phase 2: ROI 지정 반자동 추출 (전체 에셋의 20% 확보)
- **실행 대상**: 그룹 B (레퍼런스 시트 2종, 애니메이션 시트 1종)
- **우선 항목**: 애니메이션 시퀀스 프레임, 누락된 차/재료 아이콘 보충, 언어별 탭 버튼
- **작업**: Python 스크립트에 구역 좌표 입력 후 실행. 애니메이션은 행당 프레임 수(5~8개)를 고려한 미세 조정.

### Phase 3: 수동 정밀 마스킹 (전체 에셋의 10% 미만)
- **실행 대상**: 그룹 C 및 자동화 실패 항목
- **작업**: 그래픽 툴로 겹친 가구 분리, 배경 씬에서 UI 버튼 수동 분리. 이펙트 및 파티클 알파채널 복구 작업.

## 3. Unity 적용 시 크롭 주의사항

1. **원근 바닥재 처리**: 바닥 타일은 사다리꼴(원근)이 적용되어 있으므로, 자동 추출된 박스의 최하단 중앙을 Unity Sprite의 Pivot(Custom)으로 설정해야 맵 배치 시 어긋나지 않습니다.
2. **애니메이션 프레임 정렬**: 브루잉 컵 애니메이션 등은 크기가 변하더라도 기준점(컵의 하단 중앙 등)이 흔들리지 않도록 원본 시트의 간격을 유지한 채 동일한 Width/Height로 자르는 것이 유리할 수 있습니다.
3. **오브젝트 병합 위험 방지**: 선반 위에 놓인 찻잔 등 복합 오브젝트는 개별 분리하려 하면 픽셀이 훼손되므로 가급적 하나의 '통합 에셋(Set)'으로 크롭하는 것을 기본 원칙으로 합니다.
